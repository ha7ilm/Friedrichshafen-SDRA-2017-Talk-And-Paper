Hello everyone! My name is Andras. As an introduction, two years ago, I had the chance to present here about my open source project, OpenWebRX, an open source SDR application that has a web user interface. It looks like this. It allows ham radio operators to share their receiver with the world over the Internet.
Currently over a hundred stations are available online on my website, http://SDR.hu.

Motivation:

Add BPSK31/RTTY to the OpenWebRX UI
This talk will mainly focus on BPSK31 and radioteletype, which I suppose that you're all familiar with. So why did I want to implement demodulators for these modes? If someone wanted to decode digital modes, he needed to use an external modem software like Fldigi, and virtual audio cable drivers. This is quite inconvenient, also the audio is streamed in a compressed form, so the dynamic range of the audio signal is somewhat decreased. My main motivation was to integrate these demodulators directly into OpenWebRX.

Motivation:

Add general algorithms for digital modes to CSDR
I also wanted to add make the components of my demodulator reuseable, using the same tool that does DSP processing behing OpenWebRX, which is called CSDR. CSDR allows you to build simple data flow graphs in the command line. Here you can see an example command line for a wideband FM demodulator. Different processes carry out different functions, different stages of processing. When I first experimented with this approach, I was quite unsure that people will like this idea. However, last november I was contacted by someone at NASA JPL that they used this command line tool in their DSP pipeline to receive a Mars lander.
